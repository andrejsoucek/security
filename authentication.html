Authentication

Attack vectors

  • Brute force
  • Dictionary attack
  • Session management attacks

Security weakness

  • Login forms
  • Login endpoints
  • Stateful applications

Impacts

  • Identity theft
  • Money laundering
  • Sensitive information disclosure

Is the application vulnerable?

  • Application permits automated attacks - credential stuffing, brute force
  • Application permits weak and well-known passwords
  • Weak or ineffective credential recovery and forgot-password processes
  • Plain text, encrypted, or weakly hashed passwords
  • Missing or ineffective multi-factor authentication
  • Session ID exposed in the URL
  • Not rotating session IDs after login
  • Incorrect session IDs rotation (invalidation on logout)

How to prevent?

  • Implement multi-factor authentication
  • Do not use any default credentials (admin/admin)
  • Set minimum password length
  • Limit or delay failed login attempts, alert when an attack is detected
  • Use a secure session manager (random session ID with high entropy)
  • Do not put session IDs into the URL, invalidate on logout, idle and timeout
  • Password rotation and complexity requirements are viewed as encouraging users to use, and reuse, weak passwords

  • Crackable by tools
